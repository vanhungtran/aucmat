# aucmat v0.2: Statistical Inference and Multivariate Simulation

Date: 2026-07-21
Status: Approved design

## Purpose

Version 0.2 makes `aucmat` a research-grade foundation for matrix-first ROC analysis with binary outcomes. It adds coherent inference for correlated AUCs and a reproducible multivariate simulator that jointly targets marginal AUCs and biomarker dependence.

Later releases will reuse the outcome, estimand, covariance, resampling, and simulation interfaces for multivariable prediction, multiclass outcomes, survival outcomes, and out-of-memory matrices.

## Current Baseline

Version 0.1 already provides matrix-wide direction-preserving AUC screening, DeLong and bootstrap intervals, multiplicity adjustment, paired comparisons, rank stability, plots, and several simulators.

Version 0.2 addresses these observed limitations:

- `compare_auc()` supports paired DeLong only, fixes comparison intervals at 95%, and may reinterpret the positive class.
- There is no omnibus test for several correlated AUCs.
- Same-data top-N selection is not identified as post-selection inference.
- Probit calibration draws fresh random samples at successive objective evaluations, making calibration noisy.
- Infeasible simulation targets lack complete requested-versus-used diagnostics.
- Simulation functions have no dedicated test suite.
- Package checking currently reports a `plot_roc_top()` documentation mismatch and vignette warnings.
- Complete-case filtering can leave retained data, masks, missingness denominators, and summary counts inconsistent.

## Scope

### Included

- A reusable binary-outcome specification.
- Full and partial ROC AUC estimands.
- Individual, global, pairwise, directional, non-inferiority, and equivalence inference.
- Paired comparisons for biomarkers measured on the same subjects.
- Optional whole-curve and fixed-operating-point comparisons.
- A joint covariance engine for correlated full AUCs.
- A shared reproducible stratified-resampling engine.
- Multivariate simulation with target AUCs and dependence.
- Supplied or generated binary outcomes.
- Simulation feasibility diagnostics and repeated validation.
- Backward compatibility for the current API.
- Statistical, regression, reproducibility, documentation, and performance tests.

### Deferred

- Fitting multivariable biomarker prediction models.
- Unpaired comparisons between unrelated cohorts.
- Multiclass AUCs and multiclass simulation.
- Time-dependent AUCs, censoring, and competing risks.
- Sparse, delayed, and on-disk matrix backends.

These require distinct estimands and validation and will be implemented as later releases.

## Architecture

### Outcome specification

Outcome normalization occurs once. The internal specification records:

- Original and normalized labels.
- Explicit positive and negative classes.
- Original row indices and optional subject IDs.
- Outcome-missing and final analysis masks.
- Positive, negative, included, and excluded counts.

The specification is stored in `aucmat_screen` and reused by comparisons, plots, stability, and validation. Downstream functions do not infer factor direction again when a fit is available.

### AUC estimand specification

The estimand records:

- Full or partial AUC.
- Raw direction-preserving AUC or descriptive strength.
- For partial AUC, sensitivity or specificity focus, interval, and standardization.
- Null value and confidence level.

Formal inference always uses `auc_raw` with the prespecified positive class and the biomarker's original numeric orientation. `auc_strength = 0.5 + abs(auc_raw - 0.5)` is a descriptive ranking quantity and is never substituted into a DeLong contrast after data-driven flipping.

For full raw AUC, chance is 0.5. For partial AUC, the null is derived from the focus, interval, and standardization; it is not automatically 0.5. Automatic ROC direction selection is disabled in oracle comparisons.

### Covariance and inference

The full-AUC engine returns point estimates, DeLong placement values, variances, and the joint covariance matrix for biomarkers on a common subject set. Inference consumes this representation rather than recomputing outcome encodings.

Partial AUC, whole-curve, and operating-point inference use resampling because they are not covered by the full-AUC DeLong covariance implementation.

### Resampling

One internal engine supplies stratified bootstrap or permutation indices for:

- Individual intervals.
- Pairwise differences.
- Partial AUCs.
- Whole-curve and operating-point comparisons.
- Rank stability.
- Simulation validation.

It has serial and parallel paths, deterministic seed streams, progress reporting, failure accounting, and a minimum-success rule.

### Simulation

The simulator separates target validation, dependence calibration, clean generation, marginal transformation, perturbation, and empirical verification. Later outcome generators can reuse the dependence and perturbation components.

## Public API

The primary functions are:

```r
aucmat()
compare_auc()
compare_auc_global()
simulate_auc_matrix()
validate_simulation()
```

Current S3 classes remain. Version 0.2 adds `aucmat_simulation` and a simulation-validation class with `print()` and `summary()` methods.

## Screening Inference

`aucmat()` continues to test each raw AUC against chance and adds:

- `alternative = c("two.sided", "greater", "less")`.
- Consistent configurable confidence levels.
- Full or partial AUC specification.
- Explicit partial-AUC focus, interval, and standardization.
- DeLong dispatch for supported full AUCs and bootstrap for partial AUCs.

Results record method, alternative, estimand, null, confidence level, sample counts, statistic, standard error when defined, interval, p-value, adjusted p-value, status, and warnings.

## Pairwise Comparisons

`compare_auc()` remains focused on biomarkers measured on the same subjects. It compares a reference with others, all pairs in a named set, or selected top biomarkers.

Supported targets are:

- Full-AUC difference.
- Partial-AUC difference.
- Whole-ROC equality through permutation.
- Sensitivity at fixed specificity or specificity at fixed sensitivity through bootstrap.

The AUC difference is always `delta = AUC_a - AUC_b`. Supported hypotheses are:

- Two-sided superiority: `H0: delta = 0`.
- Directional superiority: `H0: delta <= 0` or `H0: delta >= 0`.
- Non-inferiority for positive margin `m`: `H0: delta <= -m`.
- Equivalence through TOST: `H0_lower: delta <= -m` and `H0_upper: delta >= m`.

Equivalence reports `p_equiv = max(p_lower, p_upper)` and the corresponding `1 - 2 * alpha` interval. A finite positive margin is required. Contradictory combinations of comparison type, alternative, and margin are rejected. Multiplicity adjustment applies to each final declared p-value, including the final TOST p-value.

Paired DeLong is the default for supported full AUCs. Stratified bootstrap is available for full AUC and required for partial AUC and fixed operating points. Whole-curve comparison uses permutation. Output always names the target and method.

### Bootstrap contract

- Subjects are sampled within outcome class.
- The same paired subject indices are applied to every biomarker in a comparison.
- The default interval is percentile.
- The default bootstrap test uses the bootstrap standard error with a normal-reference statistic and reports that choice separately from the interval method.
- Requested, successful, and failed replicate counts are returned.
- At least `max(200, ceiling(0.9 * boot_n))` successful replicates are required.
- Zero-variance, perfect-separation, and constant-marker cases return an explicit non-testable status rather than a misleading p-value.

### Missing observations and row identity

- A pairwise comparison uses subjects observed for the outcome and both biomarkers.
- A global comparison uses one common-complete set for every included biomarker.
- Every result reports common total, positive, and negative counts and removals.
- When `fit` is supplied, stored labels, original row indices or IDs, and masks are checked against `X` and `y`.
- The current `compare_auc(fit, X, y, ...)` call remains valid.
- `X` and `y` may be omitted only when the fit retained sufficient aligned data.
- Without IDs, exact length and stored row-order metadata are required and the limitation is reported.

### Post-selection warning

When `top_n` selects and tests biomarkers on the same observations, the result records `selection_status = "same_data"` and warns that inference is exploratory. Documentation recommends prespecification, a holdout cohort, or the v0.3 cross-validated workflow.

## Global Comparison

`compare_auc_global()` tests:

```text
H0: AUC_1 = AUC_2 = ... = AUC_p
```

for full raw AUCs on a common subject set. A contrast matrix and the joint DeLong covariance produce a Wald statistic.

The default contrasts compare AUCs 2 through `p` with AUC 1; any full-rank equivalent contrasts yield the same test. Covariance eigenvalues determine numerical rank under a documented relative tolerance. A generalized inverse is used only on the stable estimable subspace, with degrees of freedom equal to effective rank. Rank-zero or unstable cases return `status = "non_testable_singular"`.

The default safety limit is 100 biomarkers unless raised explicitly. With two biomarkers, the global statistic must equal the squared paired-DeLong z statistic within tolerance.

Results contain estimates, covariance, contrast rank, statistic, degrees of freedom, p-value, common sample counts, and singularity diagnostics. Pairwise adjustment supports BH, Holm, Bonferroni, or none.

## Multivariate Simulation

The conceptual interface is:

```r
simulate_auc_matrix(
  n = NULL,
  prevalence = NULL,
  y = NULL,
  positive = NULL,
  target_aucs,
  correlation,
  correlation_type = c("pearson", "spearman", "latent"),
  correlation_scope = c("marginal_observed", "within_class", "latent"),
  distribution = "normal",
  structure = c("user", "exchangeable", "ar1", "block"),
  feasibility = c("error", "nearest"),
  seed = NULL,
  ...
)
```

### Outcome modes

The caller supplies either `y` and `positive`, or `n` and `prevalence`; the modes are mutually exclusive. Supplied outcomes retain exact values, class counts, names, and row order. Generated outcomes default to exact rounded class counts with randomized row order; an explicit Bernoulli mode permits random counts.

### Targets and correlation meaning

- Any positive number of biomarkers.
- Direction-preserving population AUCs strictly inside `(0, 1)`, including values below 0.5.
- User, exchangeable, AR(1), or block dependence.
- Pearson, Spearman, or latent coefficients.
- An explicit correlation scope: marginal observed, within class, or latent.

Biomarker correlation is distinct from biomarker-outcome correlation. Existing `generate_auc_cor_vector()` semantics are not reinterpreted as between-biomarker correlation.

Class shifts from non-null AUCs affect marginal observed correlation. When marginal targets are requested, the simulator derives the implied within-class or latent dependence and checks the complete joint matrix. Conditional, marginal, and latent quantities are never treated as interchangeable.

### Distributions

The registry ultimately supports normal, log-normal, skewed continuous, heavy-tailed, and ordinal biomarkers. Parameters can be scalar or biomarker-specific. Monotone transforms preserve ROC AUC but can alter Pearson correlation, so nonnormal Pearson targets require additional calibration and can be infeasible.

### Default generator

The default is a latent Gaussian-copula/probit construction. Biomarker association with the latent outcome is calibrated to population AUC using deterministic integration where practical or fixed common random numbers otherwise. It does not draw a fresh unrelated sample for every objective evaluation.

### Feasibility

Validation checks dimensions, names, finite values, symmetry, unit diagonal, coefficient bounds, eigenvalue tolerance, positive definiteness, and compatibility of AUCs with the complete dependence structure.

`feasibility = "error"` stops with a structured condition and quantitative diagnostics.

For the Gaussian core, `feasibility = "nearest"` applies a Higham-style projection to the complete latent correlation matrix, minimizing Frobenius distance with unit diagonal and positive-semidefinite constraints. It does not silently prioritize AUCs over correlations. It reports requested, implied, adjusted, and achieved matrices, convergence tolerance, maximum absolute adjustment, and Frobenius adjustment.

More complex nonnormal Pearson calibration uses an explicitly named constrained objective and returns the same diagnostic categories. No matrix is silently repaired.

### Perturbations and target timing

Optional modules add batch/site effects, subject or biomarker outliers, and MCAR, MAR, or MNAR missingness. Population targets apply to clean pre-perturbation data. Clean finite-sample and final observed diagnostics are reported separately.

MAR and MNAR require an explicit missingness formula or documented parameterization. Oracle scores are returned only where a Bayes or linear-discriminant score is defined; otherwise the field is `NULL` with a status.

### Result

`aucmat_simulation` contains:

- Biomarker matrix and outcome.
- Optional subject, batch, and missingness metadata.
- Requested, implied, and used population parameters.
- Clean and observed empirical AUCs and correlations.
- Latent parameters and calibration diagnostics.
- Feasibility adjustments and warnings.
- Seed and RNG metadata.
- An oracle score only where defined.

### Existing simulation functions

Each current simulator receives regression fixtures before refactoring:

- `generate_auc_vector()` retains exact finite-sample rank-AUC semantics.
- `simulate_auc_correlation()` retains repeated conditional simulation semantics.
- `generate_auc_cor_vector()` retains biomarker-outcome correlation semantics.
- The current multivariate generators retain their documented model semantics.

A function delegates to the new engine only when semantics and return structure are demonstrably preserved. Otherwise it remains unchanged or enters a documented deprecation cycle.

### Simulator milestones

1. Gaussian latent/conditional core; supplied or generated outcome; user, exchangeable, AR(1), and block matrices; Pearson and latent targets; feasibility; repeated validation.
2. Spearman targets and nonnormal continuous marginals.
3. Ordinal biomarkers and batch, outlier, MCAR, MAR, and MNAR perturbations.

Each milestone has an independent acceptance gate.

## Simulation Validation

`validate_simulation()` repeats a specification and reports:

- AUC bias and RMSE by biomarker.
- Correlation bias, RMSE, and maximum absolute error.
- Monte Carlo standard errors.
- Target-interval hit rates.
- Calibration and generation failure rates.
- Interval coverage and Type I error when analysis is requested.
- Class-count and observed-missingness distributions.

Acceptance limits use a fixed scientific tolerance or a multiple of Monte Carlo standard error, whichever is larger. One sample is never treated as proof of population calibration.

## RNG and Parallel Behavior

With `seed = NULL`, random functions consume the caller's current RNG stream normally. With an explicit seed, execution uses a local scope and restores the prior global state on success or error, including when `.Random.seed` was previously absent.

Parallel work uses reproducible independent streams such as L'Ecuyer-CMRG. The same seed, backend, worker count, and package version reproduce results. Equality across different backends or worker counts is not promised. Serial fallback is always available.

## Errors and Warnings

Structured condition classes include:

- `aucmat_invalid_outcome`.
- `aucmat_invalid_correlation`.
- `aucmat_infeasible_targets`.
- `aucmat_insufficient_sample`.
- `aucmat_resampling_failure`.

A single unusable biomarker normally produces a row-level status. Invalid global inputs stop immediately. Resampling never returns an apparently valid result below its success threshold.

## Backward Compatibility

- Existing calls keep current defaults and return classes.
- Existing columns remain unless a documented deprecation cycle is used.
- New columns are additive.
- Existing simulator return structures and semantics are preserved.
- Deprecations remain for at least one minor release.
- Stored positive-class semantics override new inference from factor ordering.
- Complete-case filtering keeps retained `X`, retained `y`, masks, denominators, and `sample_summary$n_used` synchronized.

## Performance Boundary

Version 0.2 targets ordinary in-memory numeric matrices. Work is chunked where useful; bootstrap and validation may run in parallel. Sparse and on-disk adapters are deferred, but internal iteration must not preclude them.

Representative local benchmarks are:

- Screening at `(n = 500, p = 10,000)`.
- Paired bootstrap at `(n = 500, p = 20, B = 2,000)`.
- Multivariate validation at `(n = 1,000, p = 100)`.

On the maintained host, median runtime or peak memory should not regress by more than twofold without a documented statistical or architectural reason. CRAN tests do not enforce wall-clock budgets.

## Verification

### Numerical oracle tests

- Full AUC, DeLong standard error, interval, and paired difference agree with `pROC` within documented tolerance.
- Outcome levels and ROC direction are fixed explicitly in oracle calls.
- Partial-AUC fixtures cover both focus types, standardized and unstandardized values, ties, and the proper chance reference.
- Operating-point bootstrap behavior is checked independently where available.
- Global contrasts match direct matrix algebra.

### Edge cases

- Ties and ordinal scores.
- Missing outcomes and biomarkers.
- Reversed biomarker direction and unusual factor ordering.
- Constant and nearly constant markers.
- Perfect and nearly perfect AUC.
- Small and imbalanced classes.
- Duplicate biomarkers and singular covariance.

### Statistical calibration

- Monte Carlo-aware Type I error and power checks.
- Global-test invariance to biomarker order.
- Global reduction to squared paired-DeLong z at `p = 2`.
- Analytic non-inferiority and TOST boundary fixtures.
- Correct one-sided nulls, final equivalence p-value, and `1 - 2 * alpha` interval.

### Simulation properties

- Multiple AUC vectors, prevalences, structures, correlation types/scopes, and distributions.
- Positive and negative directions.
- Feasible and deliberately infeasible inputs.
- Supplied-`y` alignment and exact/Bernoulli class-count modes.
- Matrix names, symmetry, diagonal, finite values, eigenvalue tolerance, and strict endpoints.
- Requested, implied, adjusted, and achieved diagnostics.
- Clean and perturbed results.
- Bias, RMSE, correlation error, and failures across repetitions.

### Reproducibility

- Explicit seeds reproduce output and restore present or absent global RNG state.
- Error exits restore state.
- Parallel calls satisfy the documented contract.
- Unseeded calls consume the global stream and are tested separately.

### Regression and package quality

- Every current exported simulator gets fixtures before refactoring.
- Every exported function and S3 method has regression coverage.
- Documentation examples and vignettes build.
- `plot_roc_top()` documentation matches code.
- CI covers minimum supported R, release, and R-devel, with cross-platform release jobs.
- The built tarball passes `R CMD check --as-cran` with zero errors, warnings, and notes on the maintained release-check environment with suggested packages installed.
