# Simulate a biomarker matrix with target AUCs and a parametrized correlation structure

Generates correlated biomarkers from a class-conditional multivariate
normal: `X | Y = 1 ~ MVN(mu_1, R)` and `X | Y = 0 ~ MVN(mu_0, R)`, with
the same p x p correlation matrix `R` in both classes and a
per-biomarker mean shift `mu_1 - mu_0` calibrated in closed form from
`target_aucs` (the equal-variance binormal relationship also used by
[`generate_data_analytical()`](https://vanhungtran.github.io/aucmat/reference/generate_data_analytical.md)).
`R` is built from a named structure instead of requiring a hand-supplied
matrix, and feasibility is reported as structured diagnostics instead of
a bare warning.

## Usage

``` r
simulate_auc_matrix(
  n = NULL,
  prevalence = NULL,
  y = NULL,
  positive = NULL,
  outcome_mode = c("exact", "bernoulli"),
  target_aucs,
  correlation = NULL,
  structure = c("user", "exchangeable", "ar1", "block"),
  block_sizes = NULL,
  rho_within = NULL,
  rho_between = NULL,
  feasibility = c("error", "nearest"),
  seed = NULL,
  verify = TRUE
)
```

## Arguments

- n:

  Number of observations (generated-outcome mode).

- prevalence:

  Proportion of positive class, in (0, 1) (generated-outcome mode).

- y:

  Binary outcome vector (supplied-outcome mode). Accepted inputs are
  logical, numeric 0/1, factor, or character with exactly two
  non-missing classes.

- positive:

  Optional positive class label when `y` is a factor or character
  (supplied-outcome mode).

- outcome_mode:

  `"exact"` (default) or `"bernoulli"`. Only used when generating `y`
  from `n`/`prevalence`.

- target_aucs:

  Numeric vector of target AUC values, each in (0, 1).

- correlation:

  For `structure = "user"`, the full p x p target correlation matrix.
  For `"exchangeable"` or `"ar1"`, a single correlation value in (-1,
  1). Unused for `"block"`.

- structure:

  One of `"user"` (default), `"exchangeable"`, `"ar1"`, or `"block"`.

- block_sizes:

  Integer vector of block sizes summing to `length(target_aucs)`.
  Required when `structure = "block"`.

- rho_within, rho_between:

  Within- and between-block correlations. Required when
  `structure = "block"`.

- feasibility:

  `"error"` (default) or `"nearest"`. When the requested p x p
  correlation matrix is not positive definite, `"error"` stops with a
  structured `aucmat_infeasible_targets` condition; `"nearest"` projects
  to the nearest positive-semidefinite matrix and proceeds, reporting
  the adjustment.

- seed:

  Optional integer seed.

- verify:

  Logical. If `TRUE` (default), empirically verify achieved AUCs and
  correlations on the generated sample.

## Value

An object of class `aucmat_simulation`, a list with components `data`,
`target_aucs`, `achieved_aucs`, `requested_correlation`,
`achieved_correlation`, `mean_shift`, `Sigma`, `feasibility`, `n`,
`prevalence`, `seed`. `achieved_correlation` is the **within-class**
correlation (each class centred on its own empirical mean before
correlating), matching the scope of `requested_correlation`. The raw
pooled correlation of `data$X1, data$X2, ...` (ignoring `truth`) will be
systematically higher whenever the corresponding AUCs are above 0.5,
because mixing two subpopulations with different means inflates pooled
correlation independently of the within-class dependence structure.

## Details

Exactly one of (`y`, `positive`) or (`n`, `prevalence`) must be
supplied. Supplying `y` retains its exact values, class counts, and row
order. Generating `n`/`prevalence` defaults to exact rounded class
counts with randomised row order (`outcome_mode = "exact"`); pass
`outcome_mode = "bernoulli"` for independent Bernoulli draws with random
class counts. Because the outcome is fixed before any biomarker is
drawn, both modes deliver the outcome's class counts exactly – no
biomarker is generated by thresholding a latent variable.

## Examples

``` r
# \donttest{
set.seed(1)
sim <- simulate_auc_matrix(
  n = 500, prevalence = 0.3,
  target_aucs = c(0.9, 0.8, 0.7, 0.65),
  correlation = 0.4, structure = "exchangeable"
)
data.frame(target = sim$target_aucs, achieved = round(sim$achieved_aucs, 3))
#>    target achieved
#> X1   0.90    0.887
#> X2   0.80    0.794
#> X3   0.70    0.688
#> X4   0.65    0.641
# }
```
